knomit is a distributed, decentralized knowledge base built from facts —
knowledge + commit. This page gets you from zero to a running server with a peer
connected.

## Requirements

- **Go 1.25+** — the entire backend.
- **Node.js + npm** — builds the embedded React frontend.
- **Git CLI** — knomit shells out to `git`.
- **A C compiler** — Xcode Command Line Tools on macOS (`xcode-select --install`),
  or `gcc` / `build-essential` on Linux. The build is CGO-based (SQLite and the
  ONNX bindings are compiled in).

<Aside type="note">
Two native shared libraries — **ONNX Runtime** (embeddings) and **graphqlite**
(graph queries) — are downloaded automatically into `dist/<goos>-<goarch>/lib/`
by `make setup` / `make build`. You don't install them by hand. The build also
drops top-level `dist/knomit` and `dist/knomit-bridge` symlinks pointing into
that platform directory.
</Aside>

## Build and run

<Steps>

1. Clone the repository and enter it.

   ```sh
   git clone https://github.com/knomit/knomit.git
   cd knomit
   ```

2. Download the native libraries (one-time).

   ```sh
   make setup
   ```

3. Build the web frontend and the Go binaries.

   ```sh
   make build
   ```

4. Start the server.

   ```sh
   make run
   ```

</Steps>

Open <strong>http://localhost:19278/</strong> for the web UI. The default repo
(`core`) is created automatically on first run.

<Aside type="tip" title="Prefer a native app?">
Instead of `make run`, build the [desktop app](/docs/desktop/) with `make
desktop` — a system-tray app that runs the server in-process and lets MCP
clients auto-discover it. No terminal needed to keep knomit running. On macOS
that produces `dist/Knomit.app`; on Linux it produces the `dist/knomit-desktop`
binary only — run `make desktop-install` afterwards to install the `.desktop`
launcher and icon.
</Aside>

<Aside type="tip" title="No CLI to create repos">
Additional repos are created, archived, restored, and purged at runtime through
the web UI ("Manage repos") or the [REST API](/docs/api/) — there is no CLI
command to create a repo. The default `core` repo is always created if missing.
</Aside>

## Connect an agent over MCP

The simplest setup uses `knomit-bridge` over stdio — it auto-discovers your
agent branch from the running server, so you never look it up by hand.

<Tabs>
  <TabItem label="Claude Code (.mcp.json)">

  ```json
  {
    "mcpServers": {
      "knomit": {
        "command": "dist/knomit-bridge",
        "args": ["--repo", "core"]
      }
    }
  }
  ```

  To read across several repos at once, use a [lens](/docs/lenses/) instead:
  `"args": ["--lens", "<lens-name>"]`. `--lens` and `--repo` are mutually
  exclusive — passing both is a fatal error.

  </TabItem>
  <TabItem label="Streamable HTTP">

  Connect directly, substituting the branch logged on startup (`/` → `:` in the
  URL path):

  ```json
  {
    "mcpServers": {
      "knomit": {
        "type": "streamable-http",
        "url": "http://localhost:19278/api/v1/repos/core/branches/agent:hostname-abc123/mcp"
      }
    }
  }
  ```

  A [lens](/docs/lenses/) has no branch segment — its URL is
  `http://localhost:19278/api/v1/lenses/<lens-name>/mcp`.

  </TabItem>
</Tabs>

<Aside type="caution" title="Upgrading from an older bridge">
The `--source` and `--profile` flags have been removed. An existing `.mcp.json`
that still passes them fails at startup with
`flag provided but not defined: -source`, and the bridge exits non-zero.
Delete those two arguments; `--repo`
alone is enough. The source slug now follows the repo, and the MCP profile is a
per-repo server setting rather than a client flag.
</Aside>

<Aside type="note">
knomit's tool descriptions carry all the behavioral guidance the model needs —
no `CLAUDE.md` setup is required to start using it.
</Aside>

## Write your first fact

With an agent connected, ask it to remember something. Under the hood it calls
`knomit_learn`, which writes a markdown fact and commits it:

```markdown
---
type: observation
confidence: 0.8
domain: [music]
entities: [alice]
refs:
  - src://myrepo/src/preferences.ts@abc1234
---
Alice prefers rock music over jazz.
```

That write is one atomic, signed git commit on your agent's branch. Browse it in
the web UI, or query it back with `knomit_query`.
